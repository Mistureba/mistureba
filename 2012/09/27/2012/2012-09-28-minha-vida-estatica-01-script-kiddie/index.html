Minha Vida Estática #01 – Script Kiddie

Hoje começarei a fazer relatórios sobre minha experiência com um Blog WordPress Estático (a.k.a., uma versão html do blog extraída de uma instalação do WordPress no seu próprio computador). Isso servirá de ajuda para quaisquer outras pessoas interessadas no assunto.

Primeiramente, pra quê um site estático? Bem, serviços de hospedagem com os requisitos necessários para rodar o WordPress são relativamente baratos, mas um pico de visitação pode fazer seu site ficar lento ou mesmo ser bloqueado temporariamente pela hospedagem, pois – a grosso modo – o WordPress precisa consultar uma tabela/cache com o conteúdo das páginas, renderizar a página e então mandá-la ao visitante toda vez que ocorre uma visita.

No caso de páginas estáticas, as etapas iniciais (consulta à tabela/cache e renderização) já foram realizadas, restando apenas enviar a página ao visitante a cada visita. Isso dá mais trabalho do que simplesmente usar um WordPress hospedado remotamente, mas o ganho em velocidade obtido pelo site pode valer mais a pena do que pagar por uma hospedagem mais potente.

Instalação Local

O primeiro e maior empecilho para uma pessoa que pretenda utilizar páginas estáticas extraídas de um blog WordPress instalado localmente é … o próprio WordPress Local.

Serei franco com vocês: o WordPress não foi feito para rodar localmente sem ser na forma de um ambiente de testes. Instalar um WordPress localmente para checar como fica aquele super-pacote de Temas/Plugins é uma coisa, produzir seu Blog localmente é outra.

Existem muitos pacotes capazes de instalar um WordPress localmente (o que estou usando agora se chama InstantWP), mas isso significa que você deverá ter cuidados redobrados com configurações e backups – atualmente, eu faço um backup diário de TODA a minha instalação do InstantWP, assim caso a instalação deixe de funcionar, basta restaurar a versão anterior.

Outra coisa importante é evitar fazer testes na sua instalação de produção do WordPress. Assim, você evita ter que apelar para o seu Backup toda vez que algo der errado.

Convertendo o Blog

Para essa tarefa, eu usava o WP Static HTML Output (me recuso a por o link por que o autor do plugin não está nem aí pros seus bugs), porém acabei descobrindo que ele não funciona corretamente numa instalação Local, de qualquer maneira.

Atualmente, estou usando os programas Wget e FART para essa tarefa. Existem muitos tutoriais de uso dessas ferramentas pela internet afora, mas acredito que esta seja a primeira vez que alguém vai fazer um tutorial de como utilizá-las para gerar um site WordPress estático:


– Primeiramente, este tutorial leva em conta que você possui uma instalação local do WordPress rodando em uma máquina Windows. Checado? Bola pra frente;


– Baixe o Wget e o FART;


– Escreva um arquivo batch (de extensão .bat) com a seguinte linha de comando:


“” -m ““


“” -r -i -C *.html “” ““


As minhas linhas de comando ficam assim, por exemplo:


“D:Wgetwget.exe” -m “h**p://127.0.0.1:4001/wordpress/“


“D:Fartfart.exe” -r -i -C *.html “hp://127.0.0.1:4001/wordpress/” “hp://masmorradescuidada.com/blog/“


A primeira linha manda o Wget fazer uma cópia do seu site WordPress local e a segunda linha manda o Fart substituir o link de seu WordPress local pelo link de seu site remoto dentro dos arquivos copiados.


– Dê duplo clique no arquivo Batch criado, isso vai fazer uma tela preta aparecer com o Wget fazendo o serviço seguido pelo Fart dando o toque final. Quando ele terminar, a tela fechará sozinha. O resultado final estará na mesma pasta em que você criou o arquivo Batch.

Neste ponto, você deve ter um monte de arquivos HTML com link’s apontando para sua localização final, prontos para serem subidos no seu servidor.

Um adendo importante é que se seu Tema fizer uso de imagens e/ou Javascript, você deverá subir os arquivos de seu Tema manualmente até a pasta “wp-content/themes” do seu site estático. Eu uso um truque para evitar repetir esse trabalho, mas irei explicar como fazer isso no próximo relato, junto com outros truques.
